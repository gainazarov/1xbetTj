# Документация и инструкция по работе с Telegram-ботом 1xBet TJ

Дата: 10 января 2026 г.

GitHub-репозиторий проекта: https://github.com/gainazarov/1xbetTj

---

## 1. Описание проекта

Проект — Telegram-бот на **aiogram 3**, который выполняет две ключевые задачи:

1) **WebView (Telegram WebApp):** ведёт пользователей в веб‑сервис внутри Telegram.
2) **Коммуникации и аналитика:** позволяет администраторам делать массовые рассылки из постов Telegram‑канала и смотреть статистику аудитории/рассылок.

Бот **не заменяет сайт**, а является «точкой входа» и каналом уведомлений.

---

## 2. Роли и права доступа

### 2.1 Обычный пользователь
Обычный пользователь может:
- запустить бота (`/start`);
- открыть сервис в WebView через кнопку «Открыть сервис»;
- пользоваться кнопкой «Играть» (WebView);
- получать рассылки.

### 2.2 Администратор
Администратор определяется списком `ADMIN_IDS` в файле `.env`.
Администратор может:
- открыть админ‑панель;
- создавать рассылки;
- планировать рассылки на будущее время;
- просматривать и отменять запланированные рассылки;
- смотреть статистику.

Важно: обычные пользователи не видят и не могут вызвать админ‑действия.

---

## 3. Пользовательский интерфейс бота

### 3.1 Стартовое сообщение (`/start`)
После команды `/start` бот:
- регистрирует пользователя в базе (`bot.db`);
- показывает кнопку «Открыть сервис»;
- если пользователь является админом — дополнительно сообщает о доступе к админ‑функциям.

### 3.2 WebView
- **Кнопка:** «Открыть сервис» (inline).
- После нажатия бот показывает reply‑клавиатуру:
  - для пользователя: «Играть»;
  - для админа: «Играть» + «⚙️ Админ‑панель».

При нажатии «Открыть сервис» фиксируется событие WebView (таблица `webview_events`).

---

## 4. Сценарии для обычного пользователя (клиента)

### Сценарий 1: Первый запуск и вход в сервис
1) Пользователь открывает чат с ботом.
2) Отправляет команду `/start`.
3) Нажимает кнопку **«Открыть сервис»**.
4) Нажимает reply‑кнопку **«Играть»**.
5) Telegram открывает сайт в WebView по адресу `SITE_URL`.

Ожидаемый результат: сервис открывается внутри Telegram, без внешнего браузера.

### Сценарий 2: Получение новостей/акций/уведомлений
1) Пользователь ранее запускал бота (есть в базе).
2) Администратор запускает рассылку.
3) Пользователь получает сообщение‑копию поста канала.

Ожидаемый результат: пользователь получает пост (текст/медиа/кнопки), как в канале.

## 5. Сценарии для администратора

### 5.1 Подготовка перед первой рассылкой (один раз)
1) Добавьте бота **администратором** в Telegram‑канал, откуда будут браться посты.
2) Дайте права, позволяющие боту читать посты канала.
3) Убедитесь, что в `.env` заполнены:
   - `BOT_TOKEN` — токен бота;
   - `SITE_URL` — URL WebView‑сервиса;
   - `ADMIN_IDS` — ваш Telegram ID (и при необходимости других админов);
   - `DB_PATH` и `LOG_FILE` (если используете нестандартные пути).
4) Запустите бота.

Важно (ограничение Telegram): бот видит **только те посты канала**, которые появились **после** того, как бот был добавлен админом и запущен.

---

### 5.2 Открытие админ‑панели
Есть два способа:

**Способ А (inline):**
1) Нажмите «Открыть сервис».
2) Нажмите кнопку «Админ‑панель».

**Способ Б (reply):**
1) Нажмите «Открыть сервис».
2) Внизу появится reply‑клавиатура.
3) Нажмите «⚙️ Админ‑панель».

В админ‑панели доступны пункты:
- «Создать рассылку по ссылке»
- «Создать рассылку из постов»
- «Статистика»
- «Запланированные рассылки»
- «Закрыть»

---

### 5.3 Сценарий: Рассылка по ссылке на пост (рекомендуемый способ)
**Когда использовать:** когда у вас уже есть ссылка на нужный пост канала.

1) Откройте админ‑панель.
2) Выберите **«Создать рассылку по ссылке»**
   (или нажмите reply‑кнопку **«📨 Рассылка по ссылке»**).
3) Отправьте боту ссылку на пост в формате:

```text
https://t.me/channel_name/123
```

Также поддерживаются приватные ссылки вида:

```text
https://t.me/c/123456789/123
```

4) Бот показывает превью поста (копирует сообщение из канала в чат администратора).
5) Выберите **тип рассылки**:
   - Новости
   - Акция
   - Важное
   - Тест (только админы)
6) Выберите действие:
   - **«Отправить сейчас»** — рассылка пойдёт в фоне, позже придёт отчёт.
   - **«Запланировать»** — бот попросит дату/время.
   - **«Отмена»** — отменяет создание рассылки.

Ожидаемый результат:
- при отправке сейчас: бот сообщает, что рассылка пошла в фоне, затем присылает итог (получателей/доставлено/ошибки);
- при ошибке доступа к посту: бот просит проверить, что он добавлен админом в канал и ссылка верная.

Как отменить на этапе ввода ссылки:
- нажмите inline‑кнопку «Отмена» или
- отправьте текст `отмена` / `cancel` / `/cancel`.

---

### 5.4 Сценарий: Рассылка из списка сохранённых постов (вариант A)
**Когда использовать:** когда нет готовой ссылки или нужно выбрать из последних публикаций.

1) Откройте админ‑панель.
2) Выберите **«Создать рассылку из постов»**
   (или reply‑кнопку **«📨 Рассылка из постов»**).
3) Бот покажет список до 10 последних постов, которые он успел сохранить.
4) Выберите нужный пост.
5) Бот покажет превью и предложит выбрать тип рассылки.
6) Далее — «Отправить сейчас» или «Запланировать».

Если список пуст:
- убедитесь, что бот добавлен админом в канал;
- опубликуйте несколько новых постов после добавления бота;
- повторите попытку.

---

### 5.5 Сценарий: Планирование (отложенная отправка)
1) При создании рассылки нажмите **«Запланировать»**.
2) Введите дату/время в формате:

```text
ДД.ММ.ГГГГ ЧЧ:ММ
```

Пример:

```text
26.12.2025 14:30
```

3) Бот проверит формат и что время в будущем.
4) Бот создаст задачу в `scheduled_mailings` со статусом `pending`.

Часовой пояс планировщика: `Asia/Dushanbe`.

---

### 5.6 Сценарий: Просмотр и отмена запланированных рассылок
1) Откройте админ‑панель.
2) Выберите **«Запланированные рассылки»**
   (или reply‑кнопку **«🕒 Запланированные рассылки»**).
3) Бот покажет последние 10 задач (ID, тип, время, статус).
4) Для задач со статусом `pending` появятся кнопки **«Отменить ID …»**.
5) Нажмите нужную кнопку — статус станет `cancelled`, задача не будет выполнена.

---

### 5.7 Сценарий: Просмотр статистики
1) Откройте админ‑панель.
2) Нажмите **«Статистика»** (или reply‑кнопку «📊 Статистика»).

Бот покажет:
- аудитория: всего, новые за 24 часа, активные за 24ч/7д/30д, удалили бота;
- последние рассылки: тип и отчёт «доставлено/получателей/ошибок».

Примечания по реализации:
- клики по кнопкам и CTR не считаются;
- событие WebView фиксируется в момент нажатия «Открыть сервис».

## 6. Техническая информация (для владельца/админа сервера)

### 6.1 Основные настройки (.env)
Пример:

```bash
BOT_TOKEN=1234567890:AA...your_token_here
SITE_URL=https://example.com
ADMIN_IDS=111111111,222222222
DB_PATH=bot.db
LOG_FILE=bot.log
```

Ключевые параметры:
- `BOT_TOKEN` обязателен;
- `ADMIN_IDS` — список Telegram ID админов.

### 6.2 Где хранятся данные
- База SQLite: `bot.db`
- Логи: `bot.log` (с ротацией)

Основные таблицы: `users`, `mailings`, `scheduled_mailings`, `webview_events`, `channel_posts`.

### 6.3 Как работает планировщик
Фоновая задача проверяет запланированные рассылки каждые ~30 секунд.
Статусы задач: `pending`, `processing`, `done`, `failed`, `cancelled`.

## 7. Обновление на сервере (production)
Если бот поднят как systemd‑сервис, ориентируйтесь на инструкции в документе `DEPLOY_UPDATE.md`.
Кратко:

```bash
cd /opt/1xbetTj
sudo systemctl stop 1xbet-bot.service
git pull
source venv/bin/activate
pip install -r requirements.txt   # если менялся requirements.txt
deactivate
sudo systemctl restart 1xbet-bot.service
sudo systemctl status 1xbet-bot.service
```

## 8. Типовые проблемы и решения

### 8.1 «Не удалось получить публикацию» при рассылке
Причины:
- бот не администратор канала;
- нет прав читать посты;
- ссылка неверная;
- пост из приватного канала, а бот не имеет доступа.

Решение:
- добавить бота админом в канал;
- проверить ссылку и формат;
- повторить попытку.

### 8.2 В списке постов пусто (вариант A)
Причина:
- бот сохраняет только посты, полученные после добавления и запуска.

Решение:
- опубликовать новые посты после добавления бота;
- открыть меню «Рассылка из постов» снова.

### 8.3 Рассылка идёт медленно
Это нормально: бот соблюдает лимиты Telegram (паузы и обработка `retry_after`).

### 8.4 «BOT_TOKEN не задан» при запуске
Причина: нет `.env` или не заполнен `BOT_TOKEN`.
Решение: создать/обновить `.env`.
